# UMI-RL 方法选型分析

> 目的:为"在 UMI 预训练 flow-matching 策略上做 RL 后训练"确定方法路线,并记录**为什么选 / 为什么不选**的理由,供日后回看与他人复核。
> 关联:[[强化学习后训练 主题地图]]、`docs/CONTEXT.md`(术语与三家族)、`concepts/Advantage-conditioned 微调.md`。

---

## 1. TL;DR 结论

**两层结论:**

**(1) 失败模式拆两半,RL 只治一半。** 任务的失败是"泛化不够",但拆开看是两件事:**(a) 广泛泛化**(没见过的书/场景/光照就崩)——RL 治不了,只能靠数据(UMI 多样 demo + 表征),**本项目当前不碰**;**(b) 协变量漂移/无法恢复**(熟场景下漂出 demo 成功轨迹就回不来)——这才是 RL/HIL 的靶子。近期范围:**先锁窄场景把 (b) 做稳**。

**(2) 对 (b) + flow 架构 + 工程优先,第一步不是 RL,是 HG-DAgger。** flow 策略做 RL 的全部复杂度(QAM/Flow-SDE/critic)都是为"likelihood 不可解"付的税;HG-DAgger 的更新就是 **BC**(人接管纠正数据 + 现成 flow loss),**完全绕开这个税**,而 (b) 正是 DAgger 被发明出来解决的问题。因此:

- **Phase 0(第一步)**:**HG-DAgger 回路** —— 窄场景跑策略、人 gated 实时接管纠正、聚合数据用 flow loss 微调。**此阶段不需要 reward model。**
- **Phase 1(触发式升级)**:**advantage-conditioned RL** —— 仅当①想脱离人、让策略在自采 rollout 上自改进,或②BC-on-纠正触顶时才上。届时要 reward/value 模型(用 Phase 0 攒的 HIL 数据训)。
- **Phase 2(再触顶)**:Q-chunking critic + QAM,更充分榨干稀缺在线数据。
- **备选**:世界模型想象(RISE 近亲),仅当真机数据连前面都不够时启用。

一句话:**先做最简单可能直接解决问题的 HG-DAgger,只在它证明不够时才付 RL 的复杂度税。** RL 不可替代的优势只有"脱离人自改进"和"超过 teleop 质量"——都是触发式的,非起点。

## 2. 约束盘点(决策依据)

| # | 约束 | 取值 | 对选型的影响 |
|:--|:--|:--|:--|
| C1 | 预训练策略架构 | **flow-matching** | action log-likelihood 不可解 → 砍掉所有依赖 token-likelihood 的方法 |
| C2 | 真机交互预算 | **少量(几十条)** | 砍掉吃数据的纯在线 RL;偏向离线 + 少量在线、或把数据摊进模型 |
| C3 | 仿真器 | **不确定** | sim-based 方法暂不可依赖 |
| C4 | reward / 进度信号 | **未定** | HG-DAgger 阶段不需要;推迟到 Phase 1 触发时再训(用 HIL 数据) |
| C5 | 项目定位 | **工程优先,论文最后** | 目标函数 = 最稳、最可能真跑出提升;新颖性不是约束 |
| C6 | 失败模式 | **(a) 广泛泛化 + (b) 协变量漂移** | (a) 靠数据不碰;(b) 是 RL/HIL 靶子 → 近期先锁窄场景做稳 (b) |
| C7 | HIL 硬件 | **有机械臂 + 遥操实时接管** | HG-DAgger / HIL-SERL 就绪;人可在回路提供纠正动作 |

> C4 是空的,且被 C1–C3 进一步收紧:flow + 几十条 + 无 sim,意味着 reward 既不能靠 sim 给、也不能靠海量在线采样自然涌现,只能**主动设计/学习**一个打分器。

## 3. 候选方法全景:三家族

flow 策略 RL 的核心矛盾——`log π(a|s)` 不可解,标准 PPO/GRPO 的 importance ratio 算不出来。绕过它的三条路:

| 家族 | 怎么绕过 likelihood | 代表 | 天然信号来源 | arXiv |
|:--|:--|:--|:--|:--|
| **(A) 去噪当 MDP/SDE** | 把逐步去噪 reparameterize 成 SDE,每步做 policy gradient | π_RL(Flow-SDE)、RL-100 | 在线 / 仿真 | 2510.25889 / 2510.14830 |
| **(B) critic 梯度蒸馏** | 把 critic 梯度经 adjoint matching 转成对 flow 每步的逐步监督 | QAM、LWD | offline-to-online | 2601.14234 |
| **(C) advantage-conditioned 监督** | 把 advantage 当条件,退化成监督式 BC,彻底躲开 likelihood | RECAP/RISE | 离线 / 世界模型想象 | 2511.14759 |

> 配套:Q-chunking(2507.07969)不是独立家族,而是**把 RL 放到 chunk 动作空间**的"地基"——它学的是 chunk-level critic,可以喂给 (B) 或 (C) 做策略改进。

## 4. 选型矩阵(约束 × 方法适配)

打分:✅ 适配 / ⚠️ 有条件 / ❌ 不适配。

| 方法 | C1 flow | C2 几十条 | C3 无 sim | C4 reward 未定 | C5 工程优先 | 综合 |
|:--|:-:|:-:|:-:|:-:|:-:|:--|
| **HG-DAgger** | ✅ **就是 BC,用现成 flow loss** | ✅ 窄场景几轮即可 | ✅ 不需要 sim | ✅ **不需要 reward** | ✅ **复杂度最低** | **第一线** |
| advantage-conditioned (C) | ✅ 监督,不碰 likelihood | ✅ 纯离线+少量在线最省 | ✅ 不需要 sim | ⚠️ 需 value 打分器 | ⚠️ 比 DAgger 多一层 | **触发式升级** |
| Q-chunking + QAM (B) | ✅ QAM 专治 flow | ⚠️ 几十条够 TD 但紧 | ✅ 不需要 sim | ⚠️ 需 critic | ⚠️ critic 穿 flow 有工程风险 | **再升级** |
| 世界模型想象 (A/C) | ✅ | ✅ 摊薄稀缺数据 | ✅ 自带"想象" | ⚠️ 想象中 reward 更脆 | ❌ 建模本身高风险 | **备选** |
| π_RL 纯在线 (A) | ✅ | ❌ 吃在线数据 | ⚠️ | ⚠️ | ⚠️ | 暂不 |
| token-GRPO(SimpleVLA/TGRPO) | ❌ likelihood 不可解 | — | — | — | — | **否决** |
| 纯在线 PPO/SAC | ⚠️ 需特殊处理 flow | ❌ 吃数据 | ❌ 常需 sim | ❌ | ❌ | **否决** |
| sim-based GRPO(RLinf-VLA 等) | ✅ | ✅ | ❌ sim 不确定 | — | ⚠️ | 暂不 |

**关键读法**:除 HG-DAgger 外,所有 RL 方法都被 C4(需 reward)和 C1(flow 上做 RL 要付复杂度税)拖累。**HG-DAgger 同时绕开 C4 和 C1**——更新就是 BC、不需要 reward、不碰 likelihood,因此对 (b)+工程优先是综合最优的第一步。reward/value 信号不是先决条件,而是"想脱离人自改进时"才需要的东西(见 §5)。

## 5. 推荐路线:分阶段 + 为什么选

### 为什么先 HG-DAgger,而不直接上真机 RL

工程第一性:**先做最简单、可能直接解决问题的东西,只在它证明不够时才加复杂度。**

- **flow 上做 RL 的全部复杂度都是为"likelihood 不可解"付的税**(QAM/Flow-SDE/critic 穿去噪)。HG-DAgger 的更新就是 BC——聚合"demo + 人接管纠正动作",用**和 UMI 预训练时一样的 flow-matching loss** 微调,**完全不碰 likelihood、零新机制**。
- **(b) 协变量漂移正是 DAgger 当年被发明出来解决的问题**;你有机械臂 + 遥操实时接管,HG-DAgger 就绪。
- RL 相对 HG-DAgger 不可替代的优势只有两条,且都是**触发式**(非起点):① 想脱离人、让策略在自采 rollout 上自改进(需 reward model);② BC-on-纠正触顶或 teleop 本身做不好该纠正(书的接递/上架 teleop 大概率够好,故此条暂不成立)。
- HG-DAgger 与 advantage-conditioned 是一条谱系(后者 ≈ 前者 + 能用失败数据 + 按 advantage 加权),先做前者,以后平滑升级,代码与数据都不浪费。

### Phase 0 — HG-DAgger 回路(第一步)

**做什么**:窄场景真机跑 flow 策略 → 人 gated 实时接管纠正 → 聚合(demo + 纠正)数据 → 用 flow loss 微调 → 迭代。**此阶段不需要 reward model。**

**为什么是它**:见上;§4 矩阵里它同时绕开 C4 和 C1,综合最优。

### Phase 1 — advantage-conditioned RL(触发式升级)

**何时触发**:① 想脱离人、让策略自采 rollout 自改进;或 ② HG-DAgger 的 BC-on-纠正触顶。

**做什么**:用 Phase 0 攒下的 HIL 数据训一个 reward/value 模型(候选:学习式 value/critic、稀疏成功分类器+leave-one-out 2505.17016、progress 模型),给 rollout 标 advantage → advantage-conditioned 微调 flow 策略。

**为什么是升级而非起点**:它能用失败数据、能脱离人自改进,但需多一层 reward model + advantage 估计;在 HG-DAgger 够用前,这层复杂度不必付。`concepts/Advantage-conditioned 微调.md` + RISE 是认知储备。

### Phase 2 — Q-chunking critic + QAM(再触顶)

**做什么**:在 chunk 动作空间学 TD critic(Q-chunking,2507.07969),用 QAM(2601.14234)把 critic 梯度蒸馏成对 flow 策略的逐步监督。

**为什么是升级而非起点**:advantage-conditioned 只用到 advantage 的"排序"信息,没充分利用 bootstrapping;当它触顶、而你又想从那几十条在线数据里榨出更多时,引入 TD + critic 梯度是合理的下一步。但"chunk-level critic + flow 策略 + QAM"是**未经广泛验证的新组合**,工程风险高于 Phase 1,故不作为第一线。

### 备选 — 世界模型想象(RISE 近亲)

**何时启用**:若真机数据连 Phase 1 都喂不饱。训一个 dynamics + value,把稀缺 rollout 摊进模型里无限想象 rollout(RISE / WMPO 2511.09515 / VLA-RFT 2510.00406 / DiWA 2508.03645)。

**为什么是备选**:它能摊薄稀缺真机数据,但建一个够准的世界模型本身是大工程,且想象空间里的 reward/value 更脆(§ 主题地图 §4 已标此风险)。与 C5"工程优先、最可能跑通"相悖,故列备选。

## 6. 为什么不选(逐个否决)

| 被否决项 | 否决理由 | 触发哪条约束 |
|:--|:--|:--|
| **token-GRPO**(SimpleVLA-RL / TGRPO / RIPT-VLA 的 GRPO 部分) | 前提是 token-level likelihood 可解,flow 策略不满足,直接失效 | C1 |
| **纯在线 PPO / SAC** | 吃大量在线交互;你只有几十条,且常依赖 sim | C2 / C3 |
| **Q-chunking 作为第一线**(他人推荐的"第一优先") | Q-chunking 价值在于对"有意义的在线交互"做 TD backup;几十条数据撑不起,按其自身决策树也应先走 advantage-conditioned | C2 |
| **直接上世界模型当第一线** | 建模脆、工程量大,与"最稳、最可能跑通"相悖;降为备选 | C5 |
| **sim-based GRPO**(RLinf-VLA 等框架路线) | 依赖可用 sim,而 sim 还不确定 | C3 |
| **π_RL 纯在线 Flow-SDE 当第一线** | 架构对口(✅ C1),但本质是在线 RL,吃数据;几十条不够,留作有在线预算后的选项 | C2 |
| **直接上真机 RL(advantage-conditioned)当第一线** | 不是"否决"而是"推迟":对 (b)+flow,HG-DAgger 用 BC 就能解、且绕开 flow-RL 复杂度税;RL 的增益(脱离人自改进、超过 teleop)是触发式的,在 HG-DAgger 够用前不必付这层成本 | C1 / C5 |

> 对"他人推荐"的总评:其三家族判断与本文一致,但它(1)把"Q-chunking 第一优先"这个**条件结论**当成了已定决策;(2)更根本的是,它默认了"要做 RL",而没问"(b) 这个问题用 BC(HG-DAgger)是不是就够了"。本文的关键修正:**第一步不是 RL,是 HG-DAgger**;RL(advantage-conditioned → QAM)是触发式升级,reward model 也随之推迟。
